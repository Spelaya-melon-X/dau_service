## Расширенное описание задачи

### Контекст и суть

Представь платформу вроде Яндекс.Дзен. На ней миллионы публикаций от разных авторов. Когда пользователь листает ленту, система хочет показать на карточке автора **социальное доказательство** — сколько уникальных людей вчера интересовались этим автором. Это влияет на кликабельность и доверие к контенту.

### Два потока данных

**Поток 1 — входящие события (запись):**
Каждый клик пользователя на любую публикацию порождает событие. Клиент отправляет:
- `userId` — кто кликнул
- `authorId` — на чью публикацию кликнули

Метод `postEvent(Event event)` принимает эти события в реальном времени.

**Поток 2 — запросы статистики (чтение):**
Когда система формирует ленту рекомендаций, она спрашивает: *«дай мне DAU для вот этих авторов»*. Приходит список `authorId`, нужно вернуть для каждого количество **уникальных** пользователей, кликнувших за **вчерашний день**.

### Что значит "уникальных"?

Если пользователь `userId=5` кликнул на 10 публикаций автора `authorId=42` за вчера — это всё равно **1** уникальный пользователь для автора 42. Нас интересует охват, а не количество кликов.

### Временное окно

```
Вчера: 00:00:00 → 23:59:59
Сегодня: 00:00:00 → сейчас
```
Статистика всегда за **предыдущие календарные сутки**, не за последние 24 часа.

### Требования к качеству (production-ready)

- Потокобезопасность — события могут приходить параллельно
- Корректная работа с временными зонами (время сервера)
- Тесты, покрывающие граничные случаи

**1. Структура хранения данных**
Как ты будешь хранить события в памяти? Тебе нужно быстро отвечать на вопрос: *«сколько уникальных userId было у authorId=X за дату Y?»*

**2. Как хранить дату события?**
В момент `postEvent` нужно зафиксировать, к какому дню относится событие. Как это сделать?

**3. Как определить "вчера" в момент запроса?**
`getDauStatistics` вызывается сегодня, но должен вернуть данные за вчера. Как вычислить эту границу?

**4. Потокобезопасность**
Какие коллекции из `java.util.concurrent` помогут тебе избежать race condition?
